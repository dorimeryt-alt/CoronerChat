Анализ и актуализация

  1. Идентификация версии изделия — package.json / тег GitHub Latest.
  2. Трассировка требований → код — модули в п. архитектуры (providers, chat-app-server, web-ui).
  3. Изменение интерфейсов — синхронно обновлять таблицы маршрутов API и диаграммы.
  4. Эксплуатация — сверять руководство оператора с фактическими вкладками UI после релиза.
  5. Регрессия обновлений — проверять, что Latest не подменяется архивными тегами (см. 2.7.5+).

Краткий паспорт изделия

  • Наименование: CoronerChat
  • Назначение: мультиплатформенный клиент чата стримера (desktop Electron + локальный web UI)
  • Платформы: Twitch, VK Video Live, Kick, YouTube Live, Rutube, DonationAlerts, MemeAlerts, OBS
  • Канал поставки: GitHub Releases
  • Пользовательская документация: сайт · Wiki
