# Локализация UI

## Языки

| Код | Язык |
|-----|------|
| `ru` | Русский (база) |
| `en` | English |
| `ro` | Română |
| `de` | Deutsch |
| `fi` | Suomi |

Селект: **Настройки → Общие → Язык интерфейса**. После «Применить» UI перезагружается (`uiLocale` в app-state).

## Файлы

- `src/server/i18n/index.js` — `t()`, `normalizeLocale()`, загрузка словарей
- `src/server/i18n/locales/{ru,en,ro,de,fi}.json` — ключи строк
- Генерация/расширение: `node scripts/generate-i18n-locales.mjs`
- Копирование в `dist` при `npm run build`: `scripts/copy-i18n-locales.mjs`

## Как работает в UI

1. `renderIndexHtml(locale)` вшивает `__CC_I18N__` и `__CC_I18N_RU__`.
2. Клиент: `t(key)`, `applyI18nDom()` для `[data-i18n]` и автозамены точных русских подписей по словарю.
3. API: `POST /api/settings/locale` `{ locale }`.
